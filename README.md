# Call-centre-dashboard
Call Center Performance Dashboard Short Description An end-to-end Power BI project designed to analyze call center metrics, focusing on key performance indicators (KPIs), call volume trends, agent efficiency, and customer sentiment.

# Purpose
The purpose of this dashboard is to provide actionable insights to call center managers regarding service levels, call handling times, and resolution rates. It helps identify peak call times, agent performance bottlenecks, and areas for improving customer satisfaction based on the October 2020 data.

# Tech Stack
Data Visualization: Power BI Desktop Data Source: CSV File (processed via Power Query) Data Modeling: DAX (Data Analysis Expressions) for calculated measures and columns.

# Data Source📊 📊 📊
The raw data is provided in a CSV file, containing approximately 33,000 rows across 12 fields, including Call ID, Timestamp, State, Call Center City, Reason, Channel, Response Time, Sentiment, Call Duration, and CSAT Score.

# Features & Functionalities:- ✍🏻 Dynamic Interactive Filters: Users can filter the entire dashboard by specific date ranges, cities, and communication channels (Email, Web, Call Center, Chatbot). ✍🏻KPI Tracking: The top row displays critical metrics, including Total Calls, Total Duration (Hours/Minutes), Average Call Duration, and Response Time Percentage. ✍🏻 Intelligent Data Modeling: A dedicated Date Table was created to enable time-intelligence functions, linking to the main data via the Call Timestamp. ✍🏻 Visual Analysis: Includes a Map visual for geographical distribution, a Bar Chart for state-wise analysis, a Donut Chart for channel distribution, and a Tree Map for call reasons. ✍🏻 Multi-Page Report: The project includes a main Home Dashboard for high-level KPIs and a secondary Grid View Dashboard for granular, row-by-row data inspection.

# Practical Example of Use:-
A call center manager wants to investigate customer satisfaction in California.

They click on the California bar in the state chart. All other visuals update automatically to show that Billing Questions are the primary reason for calls in that state, and that Average Call Duration is slightly higher than the national average. They identify that most calls are coming through the Chatbot channel, allowing them to optimize the chatbot's FAQ section to reduce call volume.

🔮🔮# Future Scope & Enhancements🔮🔮:-
SQL Integration: The project currently uses a CSV file. Future iterations could connect directly to an SQL Database for real-time data refreshing. Predictive Analytics: Utilizing Power BI's AI capabilities to forecast future call volumes based on historical trends. Sentiment Analysis Improvement: Deepening the NLP (Natural Language Processing) capabilities to better categorize calls as positive, neutral, or negative automatically.
